Introduction

High performance computing can broadly be split into two categories: parallel computing and distributed computing. The former tends to be associated with a single 'big process' distributed over a number of processors. For example, this might be a large molecular dynamics (MD) simulation. Further, parallel computing tends to imply 'fine grained' parallelisation with libraries such as MPICH and tends to imply dedicated, tightly coupled hardware. Distributed computing, on the other hand, tends to be associated with lots of 'small processes' distributed over a number of processors. For example a series of MD simulations. Further, distributed computing tends to imply 'course grained' parallelisation, where individual copies of the executable are passed between loosely coupled hardware accompanied by different input parameter files.

In high performance computing a distinction is also made between 'dedicated' and 'non-dedicated' resources. Dedicated resources are typically specialised hardware whereas non-dedicated resources are associated with existing (mainly desktop) computers, where the CPU power far exceeds that normally demanded by its user to run, for example, word processing software.

A non-dedicated distributed network (grid) was set up at the ISIS facility. The first test of this network was to study in detail the control parameters of the hybrid Monte Carlo (HMC) search algorithm described in [2].

Case study

The control parameters of a search algorithm often crucially affect the efficiency (speed) of the algorithm. It is therefore important to be able to select 'good' values for these parameters. The implementation of HMC in [2] contains three such parameters. Fig. 1 shows the number of times (out of 20) that the HMC algorithm solved a given task for different values of the control parameters. On a single 1800MHz PC it would have taken approximately six months to create the data to produce this plot. On the grid this was completed in just under two weeks.


Figure 1: Shows the efficiency of the HMC algorithm in solving a given task for various values of its three control parameters (see [1,2] for details of what they mean). The colour shows the efficiency: from black (none of the 20 HMC runs were successful) to light copper colour (14 of the 20 HMC runs were successful).

The use of a distributed network, enabled us to define the control parameters of the HMC algorithm significantly better. This exercise demonstrates the potential of distributed computing for carrying out similar investigations.
